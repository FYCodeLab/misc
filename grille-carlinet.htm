SupBiotech Assessment Grid

Source: E.Carlinet - https://www.lrde.epita.fr/~carlinet/grille_supbio.html
1. Evaluation of Computer Code
Competency Not Acquired Insufficient Acquired Mastered
Design a Python software application to model a biological system using object-oriented programming (OOP).
  • Model the biological problem into classes and entity relationships.
  • Organize the code using OOP principles.
  • Disorganized and hard-to-follow code.
  • No or incorrect use of classes and objects.
  • Inability to translate the biological model into a computational model.
  • Some attempts at structuring, but code remains confusing and non-functional.
  • Basic use of classes and objects with errors.
  • Partial and inconsistent translation of the biological model.
  • Clear and partially modular code.
  • Correct application of OOP principles with minor errors.
  • Adequate translation of the biological model.
  • Well-organized, modular, and maintainable code.
  • Exemplary application of OOP principles.
  • Advanced and effective use of classes, objects, and design patterns.
  • Seamless and precise translation of the biological model.
2. Evaluation of Biological System Modeling
Competency Not Acquired Insufficient Acquired Mastered
Model and simulate a dynamic biological system; analyze and interpret the results.
  • Define a problem and formulate a question answerable through simulation.
  • Analyze, interpret, and critically assess simulation results.
  • Irrelevant or inconsistent model with vague interactions.
  • Inability to adapt existing models to new problems.
  • Results are uninterpreted or misinterpreted.
  • No identification of model limitations.
  • Basic model with some inconsistencies and partially precise interactions.
  • Partial and inconsistent adaptation of existing models.
  • Minimal interpretation with weak scientific justification.
  • Superficial recognition of model limitations.
  • Coherent model with precise interactions.
  • Adequate adaptation of existing models.
  • Correct interpretation with adequate scientific justification.
  • Identification of limitations with some improvement suggestions.
  • Well-designed and relevant model with very precise interactions.
  • Effective and pertinent adaptation of existing models.
  • Detailed and justified interpretation with good literature comparison.
  • Thorough recognition of limitations and clear improvement proposals.
3. Evaluation of Presentation and Presentation Support
Competency Not Acquired Insufficient Acquired Mastered
Present work through an oral presentation.
  • Unclear speech, lack of fluency, insufficient subject mastery.
  • Unattractive visual support with confusing information.
  • Basic speech with some hesitations, partial subject mastery.
  • Basic visual support with some errors or poorly presented information.
  • Clear and fluent speech with good subject mastery.
  • Well-structured and clear visual support with pertinent information.
  • Very clear, fluent, and engaging speech with deep subject mastery.
  • Attractive and pertinent visual support with effective use of visuals.
